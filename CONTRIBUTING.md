# Contributing to RuntimeOps

Thank you for your interest in contributing to RuntimeOps! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/runtimeops.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development: `npm run dev`

## Development

- **Framework:** Next.js 14+
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **Node version:** 18+

## Making Changes

1. Make your changes in a feature branch
2. Ensure code follows the project style
3. Run linting: `npm run lint`
4. Test your changes locally: `npm run dev`
5. Commit with clear messages
6. Push to your fork and submit a Pull Request

## Pull Request Process

1. Update README.md if needed
2. Update tests if applicable
3. Ensure all tests pass
4. Request review from maintainers

## Code Style

- Use TypeScript for all new code
- Follow naming conventions
- Write descriptive commit messages
- Keep components focused and modular

## Reporting Bugs

Use GitHub Issues to report bugs. Include:

- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable

## Feature Requests

Have an idea? Open a GitHub Issue with:

- Clear description of the feature
- Why it would be useful
- Possible implementation approach

## Security

For security vulnerabilities, email security@runtimeops.dev instead of using public issues.
